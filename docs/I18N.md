# Translation Guide

This guide covers every validex error code, the interpolation parameters each
message uses, and how to wire translations into popular i18n libraries.

## Key Structure

Validex produces two families of translation keys:

| Pattern | Purpose |
| --- | --- |
| `validation.messages.{namespace}.{code}` | Error messages |
| `validation.labels.{fieldName}` | Field labels |

The prefix (`validation`) and separator (`.`) are configurable via `setup()`.

## Integration Examples

### i18next

```ts
import i18next from 'i18next'
import { setup } from 'validex'

setup({
  i18n: {
    enabled: true,
    t: (key, params) => i18next.t(key, params),
  },
})
```

### vue-i18n / @nuxtjs/i18n

```ts
// In a Nuxt composable or plugin
const { t } = useI18n()

setup({
  i18n: {
    enabled: true,
    t: (key, params) => t(key, params),
  },
})
```

### Custom t() function

```ts
const translations: Record<string, string> = {
  // Paste the flattened keys from the template above
  'validation.messages.base.required': '{{label}} est requis',
  'validation.messages.email.invalid': '{{label}} n\'est pas une adresse email valide',
  // ...
}

setup({
  i18n: {
    enabled: true,
    t: (key, params) => {
      let msg = translations[key] ?? key
      for (const [k, v] of Object.entries(params ?? {})) {
        msg = msg.replace(`{{${k}}}`, String(v))
      }
      return msg
    },
  },
})
```
