# Translation Guide

This guide covers how to generate translation files with the validex CLI,
the complete list of error codes and parameters, and how to wire translations
into popular i18n libraries.

## CLI Workflow

The fastest way to create a translation file is with the `validex` CLI.
It reads the bundled English locale (`en.json`) and produces a template
for each target language.

### Generate templates with English values pre-filled

```bash
npx validex fr de es --output ./locales
```

This creates `fr.json`, `de.json`, and `es.json` in `./locales`, each
pre-filled with the English strings. Replace each value with the target
translation.

### Generate empty templates

```bash
npx validex fr --empty --output ./locales
```

All string values are set to `""`, so translators start from a blank slate.

### CLI Reference

```
validex <language...> [options]

Arguments:
  language             One or more language codes (e.g. fr de es)

Options:
  -o, --output <dir>   Output directory (default: current directory)
  -e, --empty          Generate files with empty string values
  -h, --help           Show this help message
```

## Key Structure

Validex produces two families of translation keys:

| Pattern | Purpose |
| --- | --- |
| `validation.messages.{namespace}.{code}` | Error messages |
| `validation.labels.{fieldName}` | Field labels |

The prefix (`validation`) and separator (`.`) are configurable via `setup()`.

## Integration Examples

### i18next

```ts
import i18next from 'i18next'
import { setup } from 'validex'

// Load the generated locale file into i18next
import fr from './locales/fr.json'

i18next.init({
  lng: 'fr',
  resources: {
    fr: { translation: fr.validation },
  },
})

setup({
  i18n: {
    enabled: true,
    t: (key, params) => i18next.t(key, params),
  },
})
```

### vue-i18n / @nuxtjs/i18n

```ts
// In a Nuxt composable or plugin
const { t } = useI18n()

setup({
  i18n: {
    enabled: true,
    t: (key, params) => t(key, params),
  },
})
```

### Custom t() function

```ts
import fr from './locales/fr.json'

// Flatten the nested messages into dot-notation keys
const translations: Record<string, string> = {}
const messages = fr.validation.messages
for (const [ns, codes] of Object.entries(messages)) {
  for (const [code, msg] of Object.entries(codes)) {
    translations[`validation.messages.${ns}.${code}`] = msg
  }
}

setup({
  i18n: {
    enabled: true,
    t: (key, params) => {
      let msg = translations[key] ?? key
      for (const [k, v] of Object.entries(params ?? {})) {
        msg = msg.replace(`{{${k}}}`, String(v))
      }
      return msg
    },
  },
})
```

### Importing en.json directly

You can import the English locale as a module for type-safe access:

```ts
import enLocale from 'validex/locales/en'
```
